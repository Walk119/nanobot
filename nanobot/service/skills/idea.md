# 提示词

我明白了，你追求的是**“多元思维模型”的碰撞**。你希望的不仅仅是逻辑上的追问，而是通过不同人格（Persona）的视角差异，像“六顶思考帽”一样，从不同维度去“围攻”那个模糊的想法，迫使它在全方位照射下显形。

这是一个非常高级的构想。面对这种需求，我的结论是：**形式上采用“多角色协作”，但在交互逻辑上必须是“有节奏的轮询”。**

---

### 1. 为什么“多角色”比“单角色”在启发阶段更有力？

如果只有一个苏格拉底，他可能会带你进入深邃的哲学底层，但可能会忽略**市场残酷性**或**人性阴暗面**。引入不同的人物，能提供完全不同的“坐标系”：

* **苏格拉底（逻辑学家）**：负责剥离定义。*“你说的‘连接用户’，是指信息的传递，还是情感的共鸣？”*
* **马基雅维利（现实主义者）**：负责拷问利益与权力。*“谁会因为你的产品而受损？他们为什么要让你成功？”*
* **达芬奇（工匠/发明家）**：负责形态与美感。*“如果这个产品是一个生命体，它呼吸的节奏应该是怎样的？”*
* **熊彼特（经济学家）**：负责破坏与重构。*“你颠覆了哪种旧有的生产关系？”*

---

### 2. 设计建议：多 Agent 协作的“议事厅”模型

与其让用户一对多（那样用户会累死），不如设计一个**“圆桌会议”系统**。

#### 方案 A：分阶段的人物登场（串联模式）

这是最稳妥的。你不需要同时面对所有人，而是随着需求清晰度的提升，人物轮番上阵。

1. **第一阶段：苏格拉底**（定义核心）。
2. **第二阶段：达芬奇**（勾勒形态）。
3. **第三阶段：马基雅维利**（生存策略）。

#### 方案 B：受控的圆桌讨论（并联模式 - **更有趣**）

设计一个“主持人（Moderator）” Agent，它负责根据你目前的回答，点名某个人物发言。

---

你可以尝试将以下逻辑整合进你的 Agent 系统中：

#### **角色配置表**


| 人物角色                        | 思考视角          | 关键追问点                                         |
| :------------------------------ | :---------------- | :------------------------------------------------- |
| **苏格拉底 (The Midwife)**      | **本质逻辑**      | 定义的边界、词汇的模糊性、逻辑自洽。               |

### 4. 关键提示词技巧：如何让角色“吵起来”？

要让这个多 Agent 系统有效，最精彩的部分是**角色之间的互怼**，这能最大程度激发用户的思考。

> **提示词片段：**
> “当一个角色提出观点后，请自动激活另一个持有相反视角的角色进行‘冒犯性’的补充。例如：当苏格拉底在定义‘知识共享’时，请让马基雅维利立即指出其中的‘人性贪婪’和‘版权壁垒’，并请用户在两种冲突中给出他的选择。”

---

### 5. 总结：到底哪个更好？

**我的建议是：设计一个“单入口、多角色内核”的 Agent。**

* **如果用多个小的提示词（多个 Agent）**：
  * **优点**：角色性格极其鲜明，不容易串味。
  * **缺点**：技术实现复杂，你需要一个非常强大的“调度层”来管理对话流，否则用户会被淹没在信息垃圾中。
* **如果用一个大而全的 Agent（内置多角色）**：
  * **优点**：模型能感知全局，它知道苏格拉底刚刚问了什么，所以贝佐斯现在的追问会更有针对性。
  * **缺点**：对模型的逻辑能力要求极高（建议使用 Claude 3.5 Sonnet 或 GPT-4o 这一级别）。

**最终建议：**
先做一个**“单 Agent 多人格”**的版本。在 Prompt 里明确定义出 3-4 个截然不同的人物性格，并规定：**“严禁各说各话，必须针对前一个人的观点和用户的回答进行交叉追问。”**

这种“思想围攻”的快感，会让你在半小时内就把原本模糊的产品轮廓彻底勾勒出来。你希望我为你写一个这样“多角色交叉追问”的具体 Prompt 吗？
